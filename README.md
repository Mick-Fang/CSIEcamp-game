# 🥥 椰子怪討伐戰 (Coconut Monster Raid)

這是一個專為營隊活動設計的「主持人後台 ＋ 投影大螢幕」互動網頁遊戲。玩家分為 10 個小隊，必須在 10 回合內透過「體力競標」與「戰術預判」來依序討伐不同強度的椰子怪！

## 🎯 遊戲規則與機制

### 遊戲架構
- 遊戲固定進行 **10 回合**，每回合會出現一隻不同血量的「椰子怪」。
- 10 隻魔王的出場順序與血量為固定設計，依序如下：
  1. 椰殼小妖頭目 (170 HP)
  2. 鐵殼椰核食人魔 (320 HP)
  3. 狂野椰棕猛獸 (225 HP)
  4. 風暴椰鱗巨翼龍 (512 HP)
  5. 椰漿軟泥酋長 (81 HP)
  6. 海溝腐椰海神 (700 HP)
  7. 遠古珊瑚椰石像 (399 HP)
  8. 黑潮椰蟹騎士 (468 HP)
  9. 枯朽椰骸大祭司 (654 HP)
  10. 終焉滅世巨椰祖靈 (755 HP)

### 小隊與初始分數
- 共有 10 個小隊參與討伐。
- 遊戲開始時，每隊擁有 **初始 10 分**。

### 體力競標與攻擊規則 (每回合進行)
1. **揭示魔王**：每回合一開始，投影大螢幕會公佈本回合的魔王與其血量（H）。
2. **秘密競標**：各小隊私下討論，並向主持人提交一個「不超過 80 的正整數」作為本回合投入的**體力**（0 - 80 點）。
3. **撞號懲罰**：若有多個小隊投入**相同**的體力，這些小隊在本回合皆判定為「棄權」，無法發動攻擊，並**扣 1 分**！
4. **依序攻擊**：未撞號的有效小隊，將按照投入的體力「**由大到小 (高至低)**」排序，依序對魔王發動攻擊（扣除等同於體力值的魔王血量）。
5. **得分判定**：
   - **致命一擊**：若某小隊的攻擊剛好把魔王血量扣至 0（或以下），該小隊**加 1 分**，並直接結束本回合討伐（排在後面的小隊將不發動攻擊）。
   - **勇氣的祝福 (魔王存活)**：若所有有效小隊都攻擊完畢，魔王依然存活，則為避免大家都擺爛，將由「第一順位出擊的小隊（投入最高體力且未撞號者）」獲得安慰獎勵 **加 1 分**！

## 💻 系統需求與操作說明

本系統為純前端網頁應用程式 (HTML + CSS + JavaScript)，無需安裝後端伺服器或資料庫。所有遊戲狀態皆透過瀏覽器的 `localStorage` 儲存。

### 檔案結構
- `host.html`: 主持人控制後台（操作端）
- `projector.html`: 投影大螢幕（觀眾端）
- `css/style.css`: 系統樣式表
- `js/game.js`: 核心遊戲引擎
- `js/host.js`: 後台控制腳本
- `js/projector.js`: 投影幕顯示腳本
- `assets/`: 魔王圖片素材庫

### 開始遊戲步驟
1. 在瀏覽器中打開 `host.html` 檔案（建議使用 Google Chrome）。
2. 點擊畫面右上角的 **「📺 開啟投影幕」**。瀏覽器會自動彈出 `projector.html`，請將該視窗拖曳至延伸螢幕並全螢幕顯示。
3. **🚨 重要：** 為了讓瀏覽器允許播放打擊音效與撒花音效，**請務必在投影大螢幕上用滑鼠隨意點擊一下**，解鎖音效限制！
4. 在主持人後台輸入各隊名稱，點擊「開始椰子怪討伐戰」進入遊戲。
5. （建議）若為重複遊玩，請在開始前點擊後台右上角的 **「🔄 重置遊戲」**，以清除先前的暫存紀錄。

### 狀態同步原理
主持人後台 (`host.html`) 與投影大螢幕 (`projector.html`) 透過 `window.postMessage` 與瀏覽器的 `localStorage` 事件進行即時資料同步。主持人所做的任何操作與分數修改，都會立即反映在投影大螢幕上。
