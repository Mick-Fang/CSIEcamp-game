# 🥥 椰子怪討伐戰 — 資訊營賽局遊戲系統

這是一個為資訊營賽局設計的網頁端即時控制與投影顯示系統。本遊戲改編自《資訊營賽局——終焉椰子討伐戰》說明書，融合了**資源規劃、風險評估與競標機制**。

系統採用**免伺服器 (Serverless) 的即時本地同步技術**，主持人可在本機後台輸入每輪的體力競標與邪惡椰子數量，投影幕畫面將會即時以 2D 卡牌揭示、動態血條、傷害震動及 Confetti 撒花動畫展示給台下的參賽勇士。

---

## 🏝️ 雙螢幕同步運作說明 (快速開始)

本專案為純前端網頁，**100% 支援離線運行**，無需安裝 Node.js 或架設伺服器。

1. **投影機連接**：將筆記型電腦連接至教室投影機，並在電腦系統設定中將顯示模式設為**「延伸螢幕」**。
2. **啟動入口**：在瀏覽器中直接按兩下打開專案根目錄的 [index.html](index.html)。
3. **開啟主持人後台**：在筆電主螢幕視窗中，點擊**「進入主持人後台」**。
4. **開啟投影幕**：將瀏覽器拖移至**投影機畫面（延伸螢幕）**，點擊**「開啟投影顯示幕」**（建議按 **F11** 鍵進入瀏覽器全螢幕）。
5. **音效授權**：點擊投影大螢幕任意處以解除瀏覽器的音效自動播放限制。
6. **開始遊戲**：主持人在後台點擊「開始椰子怪討伐戰」後，兩側網頁便會透過 `localStorage` 機制**雙向即時同步**！

## ⚔️ 遊戲核心規則摘要

本遊戲共進行 **8 個回合**，每隊每輪初始皆有 **100 點體力** 與最多 **3 顆邪惡椰子**：

### 1. 召喚與前鋒競標 (Step 1)
- 隊伍秘密提交 0-3 顆邪惡椰子及前鋒競標體力 (0-100)。
- 收集 10 隊的邪惡椰子總和，自動決定本輪召喚出的椰子魔物種類與血量。
- **前鋒篩選**：投標體力前三名進入「前鋒部隊」。若遇平手：
  - 先取分數低的隊伍。
  - 若分數也平手，主持人可於後台使用「命運大轉盤」抽籤決定（投影畫面同步旋轉）。
  - 若四組進入前鋒，怪物血量自動調整為 $\text{原始血量} \times \frac{4}{3}$。
- 前鋒隊伍將會扣除投標體力，並以「剩餘體力」對魔物造成傷害（傷害總和 = 前鋒成員剩餘體力之和）。

### 2. 前鋒討伐決戰 (Step 2)
- **討伐成功**（前鋒總攻擊力 $\ge$ 怪物血量）：怪物被擊殺，前鋒部隊小隊各獲得 **+4 分**，直接進入下一輪（跳過後援階段）。
- **殘血存活**（前鋒總攻擊力 $<$ 怪物血量）：怪物殘血存活，前鋒部隊本輪得 0 分，遊戲進入 Step 3。

### 3. 後援隊競標與背叛者 (Step 3)
- 未進入前鋒的隊伍使用剩餘體力進行第二次競標（後援投標），投標上限為其賸餘體力。
- **後援篩選**：投標體力前三名進入「後援部隊」進行補刀。
  - 若平手，邪惡椰子較少的隊伍優先進入後援。
  - 若同標且同椰子數遇到完全平手，啟用**平手縮放對策**：大於等於該門檻之小隊全部進入後援（$N_{support} \ge 3$），且怪物賸餘血量自動調整為 $\text{原剩餘血量} \times \frac{N_{support}}{3}$。
  - 未入選前鋒也未入選後援的隊伍，將自動被標記為**「背叛者」**。

### 4. 後援決戰與加分 (Step 3 戰鬥)
- 後援總攻擊力 = 後援成員二次投標後賸餘體力之和。
- **後援成功**（後援總攻擊力 $\ge$ 殘血血量）：後援小隊各獲得 **+2 分**，背叛者獲得 0 分。
- **後援失敗**（後援總攻擊力 $<$ 殘血血量）：後援小隊獲得 0 分，而**背叛者只要在當前回合有投入體力**（`bid1 > 0 || bid2 > 0`），即可獲得 **+1 分**。
